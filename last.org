几个问题。一、默认的概率分布是什么样的？均匀分布吗？二、同概率的时候感觉不如55随机一下更好？三、叠加方式确认无误了吗？


• 三个点分别说。

  1. 默认概率分布

  当前实现里没有默认均匀分布。

  在 src/probability.rs:1 里，如果总振幅平方和为 0：

  if total <= 0.0 {
      return Vec::new();
  }

  也就是说：

  没有路径贡献 / 完全抵消 -> 空分布 -> 无选择结果

  不是均匀分布。

  这更保守，但后续训练时可能需要明确策略。可选方案：

  A. 空分布：表示系统不知道，返回 None
  B. 均匀分布：在候选集合上随机/平均
  C. fallback 分布：使用先验或上一状态

  我倾向当前阶段保留空分布，因为它能暴露模型没有学到或发生完全抵消的情况，不会用均匀分布掩盖问题。

  2. 同概率随机

  你说得对。当前为了测试稳定，我做了确定性 tie-break：同概率选 ID 小的。这个适合单元测试，但不符合“选择函数”的自然含义。

  更合理的是支持两种选择函数：

  ArgmaxDeterministic：测试用，稳定复现
  ArgmaxRandomTie：实验用，同最大概率时随机选一个

  或者更进一步：

  Sample：按整个概率分布采样

  如果我们要模拟过程，我建议实现：

  SelectionMode::Argmax
  SelectionMode::RandomTie
  SelectionMode::Sample

  然后 example 里可以固定 RNG seed，既随机又可复现。

  3. 叠加方式是否确认无误

  目前 S->T 的叠加方式是我们讨论的“实数振幅相干叠加”：

  amp(t) = Σ coeff(type1_link) * output_amp(target_state, t)
  P(t) = amp(t)^2 / Σ amp(x)^2

  这点已经通过测试确认了：+1 和 -1 指向同一输出特征会完全抵消，输出空分布。

  但还有一个重要限定：现在只在 S->T 和新增的 S->S activate 中使用了这套振幅逻辑。完整过程还没全链路确认，因为：

  T->S 还没实现
  S->S 的 Type2Link::activate 已实现，但 example 还没真正用它跑阶段实验
  CompositionLearner 仍然是固定迁移逻辑

  所以准确说：

  局部叠加规则无误；
  完整 T->S->S->T 实验链路还没验证。

  我建议下一步先改选择函数，加入随机 tie 和采样；然后再做完整阶段式实验，否则现在的确定性 tie-break 会影响实验解释。
