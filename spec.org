* 目的
本项目的根本目的是实验探究如何才能模拟语言和思考的过程。

采取的实际方案是通过集合间的映射来研究对应的关系。

* 手段

** 模拟思路
将所有外部输入、输出及可观察内容统一作为特征集合 T。T 对应“声音、视觉、触觉、
符号、上下文标记”等感知或输入特征。

将内部集合 S 作为“状态”集合。S 中既有基本状态元素 s，也有 S 内部的映射对象。状态
的变换通过 S 内部的映射实现。

*** 集合 T 到 S 的映射
集合 T 到集合 S 上的映射可以认为是集合 T 中的任意元素 t 到集合 S 中某一元素 s 的
连线 t->s。而这样的一条连线可以通过集合 S 中所有元素 s 上的一个概率分布来决定
（通过一个选择函数，比如极大时对应的 s 决定了 t->s）。这样的所有连线构成一个
T 到 S 的映射。

可以发现，这样的映射完全由对应的 t->s 的概率分布决定（当然还有选择函数）。

这种“不同集合”之间的映射代表了从外部特征到内部状态的转换。S 中的状态元素不应写
入人工语义 tag。相似空间直接建立在 S 上，由状态间的邻域重叠决定。

*** S 上的相似空间
S 上需要有一个相似空间，用来表达哪些状态在某些上下文中相似。这个相似空间可以由一
组”邻域”构成：

#+begin_src text
SimilaritySpace {
  neighborhoods: Vec<Neighborhood>
}

Neighborhood {
  members: Set<S>
}
#+end_src

S 中的每个状态单独保存自己属于哪些邻域，以及它属于这些邻域的置信度：

#+begin_src text
State {
  id: SId,
  support_features: Vec<TId>,
  neighborhoods: Vec<NeighborhoodRef>,
  confidences: Vec<Number>
}
#+end_src

其中 =neighborhoods[i]= 与 =confidences[i]= 一一对应，表示该状态属于对应邻域的
置信度。Neighborhood 本身不保存置信度，因为置信度描述的是”某个状态属于某个邻域”
这条隶属关系，而不是邻域自身的全局属性。

例如，如果 a、b、c、d 在某个邻域中同时存在，则它们在这个相似空间中被认为具有相似
性。这个邻域可以是预输入的，也可以是后续动态学习出来的。

S 上两个元素的相似度由它们所属邻域的重叠直接决定，不再需要经过 T 诱导。这个相似
空间还可以伴随映射学习而演化——当系统发现两个状态在映射中经常扮演相似角色时，可以
将它们的邻域调整得更加接近。

T 上的特征不再保存邻域隶属关系；Feature 仅保留 ID 作为外部输入输出的标识。

*** 集合 S 到 S 的映射
S 中既有基本状态元素 s，又有 S 的子集到 S 的子集的连线。

S 内部的元素到元素的连线称为 1 型连线：

#+begin_src text
s1 -> s2
#+end_src

每个 S 中的基本状态 s 可以认为有一条自身到自身的 1 型连线 s->s（自环，恒等映射）。

S 的子集到 S 的子集的连线称为 2 型连线。例如：

#+begin_src text
{a,b,c} -> {d,e}
#+end_src

这个 2 型连线中包含若干 1 型连线：

#+begin_src text
a -> d
b -> d
c -> d
a -> e
b -> e
c -> e
#+end_src

每条 1 型连线带一个系数，这些系数构成概率分布或振幅分布，决定该 2 型连线实际倾向于
激活哪些目标状态。

复合映射可以是单一子集到单一子集（如“转”就是方向状态子集之间的映射），也可以是
多个子集到多个子集的映射。

训练的目标之一是确定各 1 型连线的强度（振幅系数）。S->S 的映射在运行时动态构造：当前
S 中活跃的状态在复合映射中找到对应的源子集，激活相应的 1 型连线，经干涉叠加后得到目标
子集上的振幅分布。选择函数从目标子集中选出下一个状态。

S->S 的映射本身就是 S 的一部分。

*** 集合 S 到 T 的映射
S->T 的映射代表内部状态或状态映射产生可观察输出。T 同时包含输入特征和输出特征；
例如听到声音是声音输入特征，开口说话是声音输出特征，它们都属于 T。

S->T 和 T->S 类似，也可以由目标特征上的概率分布及选择函数决定。但 S->T 不一定从
所有 S 元素上选择，而是从当前基本状态或当前 2 型连线激活出的状态子集上产生输出。

对于基本状态 s，s 可以对应许多输出特征：

#+begin_src text
s -> {t1, t2, t3, ...}
#+end_src

这些 t 上的系数构成一个概率分布或振幅分布，选择函数从中选出具体输出特征，或选出
一个输出特征子集。例如“苹果”这个状态可能输出颜色、气味、形状等特征；如果当前输出
通道是口中的声音，则只需要从声音输出特征子集中选择“苹果”这个声音特征。

对于 2 型连线，输出分布由两部分叠加得到：

1. 该 2 型连线内部 1 型连线的系数分布。
2. 被 1 型连线激活的目标状态子集到 T 的输出分布。

例如 2 型连线激活了目标子集 {c,d}，则最终输出可以由 {c,d}->t1...tn 的分布加权叠
加而来，再由选择函数在指定输出特征子集中确定实际输出。

当前默认采用实数振幅的相干叠加：1 型连线系数与目标状态到 T 的输出系数相乘，得到
该路径对输出特征的振幅贡献；所有路径先在同一输出特征上相加，再按 Born 规则平方并
归一化为输出概率。正负系数因此可以产生建设性或破坏性干涉。选择函数第一版使用极
大值选择。

*** 映射的复合
此时，可以发现，训练的时候所涉及到的 T->S->T 的过程完全可以被上面的映射复合来
“替代”。

复合可以认为是概率或振幅的叠加，这种叠加方式有不同的选择。当前优先使用相干振幅
叠加：先叠加振幅，再平方归一化为概率。

举例：

agent A 现在是面朝 X 方向，他的输入告诉他，目前方向为 X，转向方向 Y。因此，外部特
征集合 T 中有表示当前方向 X 的特征，也有表示“转”和 Y 的特征，然后会有
T->S->S->T（这里状态的转换就是 S->S），再通过 S->T 的映射确定输出方向是什么。

因此可以发现，动作其实被两个映射覆盖，一个是 S->S 的映射，代表状态的转变，另一个
是 S->T 的映射，代表状态的输出。

*** 生成元与复合规则的学习
除了学习具体的 T->S、S->S、S->T 映射之外，还需要学习一种机制，代表如何复合得到新
元素或新映射。这个机制可以理解为学习“生成元”或“复合规则”。

例如，“转a”不应该只是被记忆为一条孤立映射，而应尽量由“转”和“a”复合产生：

#+begin_src text
转 + a -> 转a
#+end_src

如果系统已经学习了”转a”，并且 S 上的相似空间表明 a、b、c、d 彼此相似，那么系统应
该能够把这个复合规则迁移到相似元素上：

#+begin_src text
转 + b -> 转b
转 + c -> 转c
转 + d -> 转d
#+end_src

这里的迁移依赖 S 上状态的相似性，即它们所属邻域的重叠度。

如果最后开始训练，可以先训练“转”和四个方向本身，然后训练“转X”方向，其中 X 只取
已经训练方向中的两个，最后测试剩下两个方向，观察系统能否通过相似空间和复合规则自
动生成未训练过的“转X”映射。

** 相关实现思路
*** 概率、振幅与选择函数
T->S、S->S、S->T 三类映射统一采用同一套最小运行规则：

1. 多条路径先贡献到目标元素上的实数振幅。
2. 同一目标元素的振幅相干相加。
3. 使用 Born 规则将振幅平方并归一化为概率。
4. 第一版选择函数使用极大值选择。

#+begin_src text
amp(x) = sum(path_amp -> x)
P(x) = amp(x)^2 / sum_y amp(y)^2
choose(P) = argmax_x P(x)
#+end_src

S->S 运行时需要先根据当前活跃的源状态子集筛选 2 型连线内部的 1 型连线。只有 source
在当前活跃子集中的 1 型连线会贡献目标振幅。

*** 映射与集合的抽象
首先需要最小化系统运行，所以不要做太多的优化。

将集合分成两大类，分别是“特征集合”T 和“状态集合”S。

特征集合到状态集合的映射就是 T -> S 的映射，这个需要训练，得到什么样的特征对应什
么样的状态。

状态集合内部的映射就是 S -> S 的映射。

状态集合内的映射需要抽象出来，用 struct {source:vec, target:vec}，默认状态集合内
的不变状态是自己到自己的 1 型连线（映射），这样可以统一。对于 2 型连线，还需要保存其
内部 1 型连线及对应系数。

状态集合上的映射复合会产生新的映射（因为状态集合上的映射实际上应该是状态集合的子
集到状态集合的子集的映射。因此他的复合就是 source 和 target 的变化。训练的就是这
个映射 source 和 target 的变化。

特征集合到状态集合的映射的复合，就是训练一个更高级的机制，如何根据 S 上的相似空
间判断两个状态是否相似，然后将它们对应的映射合在一起。比如：”转”对应
状态集合中一个 {a,b,c,d}->{a,b,c,d} 四方向变化的映射。而“a”对应一个 {a}->{a} 的
映射，复合后的符号“转a”即 {a,b,c,d}->{a}。要学习的是如何从前面两个映射复合产生
后者（这样会自然延拓到“转b”等等上），这个过程需要一个对比、相似过程以及一个延拓
的过程。
** 实验训练规范
首先训练基础的“转”和四个方向本身，然后训练“转X”方向，其中 X 只取四个方向中的两
个，最后测试剩下两个方向即可。
** 模拟使用的规范
使用 rust 语言，使用 log/trace 库，不随便 println。

*** Rust 代码规范

**** 格式与 lint
- 使用 rustfmt 默认格式，提交前运行 cargo fmt
- 代码中不允许有 warning，编译必须 clean（包括 dead_code、unused 等）
- 使用 cargo clippy 辅助检查

**** 日志
- 使用 log crate + env_logger，不允许使用 println!/eprintln!
- 库代码中合理使用 log::debug! / log::trace!，避免 info! 层级输出过多细节
- 实验脚本（examples/）可以使用 log::info! 输出结果

**** 命名
- 类型、trait、enum 使用 CamelCase
- 函数、方法、变量、模块使用 snake_case
- 避免使用缩写（除非是公认的，如 Cfg、Args）
- 枚举变体使用 CamelCase，而非 SCREAMING_CASE

**** 错误处理
- 库代码中避免 unwrap()/expect()，使用 Result 返回错误
- examples/ 中可以适当使用 ? 运算符
- 为自定义错误类型实现 Display + Error（或用 thiserror）

**** 代码风格
- 公共 API 需要写文档注释（///）
- 私有函数不需要注释，除非逻辑非常隐晦
- 优先使用迭代器而非下标循环（除非性能关键路径）
- 避免不必要的 clone()，优先用引用或 clone 语义明确的场景
- 不要写死常量（魔法数字），使用 const 定义
- 使用 derive(Debug, Clone, Copy, PartialEq) 等 derive 宏而非手动实现
- 模块中通过 pub use 控制对外暴露的 API 边界
